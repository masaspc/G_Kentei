## 1. プロジェクト概要

### 1.1 目的

G検定2026 #4の合格を最優先目標とし、個人の学習を最大効率化する自分専用学習プラットフォームを構築する。あくまで個人利用を前提とし、自分のペースで学習データを蓄積・分析し、AI機能はAnthropic社のClaude APIを活用して実現する。

### 1.2 背景

- G検定は2024年シラバス改訂で生成AI・倫理・法規制が大幅強化され、市販問題集と本番試験のギャップが拡大している
- 1問あたり約41秒という時間制約に対応するには、反復演習とチートシートの整備が必須
- 既存の無料サービスは存在するが、自分の学習履歴・弱点分析・記憶定着スケジューリングを統合した環境は限定的
- 自前構築により、学習データを自分で管理し、Claude APIによる高品質なパーソナライズ機能を組み込める
- Claude APIを使うことで、自分のサーバでLLMを運用する手間とコストを完全に省ける

### 1.3 スコープ

**対象範囲**

- 一問一答型の問題演習機能
- 模擬試験 (本番形式) 機能
- 学習履歴・弱点分析ダッシュボード
- 間隔反復学習 (SRS: Spaced Repetition System)
- 用語集・チートシート機能
- Claude APIによる解説生成・質問応答

**対象外**

- 多人数同時利用・他ユーザーとの共有機能
- 課金・決済機能
- モバイルネイティブアプリ
- E資格対応
- 複数ユーザーアカウント (シングルユーザー前提)

### 1.4 利用者

本サイトは個人利用専用とする。

| 区分 | 人数 | 想定利用シーン |
| --- | --- | --- |
| 利用者 | 1名 (山城) | 通勤・休憩・自宅学習。スマホとPCの両方で利用 |

※ 認証機構は実装するが、自分専用とすることでセキュリティ・運用負荷・開発工数を最小化する。

### 2.2 機能詳細

#### F-02: 一問一答演習

**概要**: 1問ずつ表示し即座に正誤判定・解説表示する演習モード

**入力**

- 出題範囲フィルタ (シラバス分野: 21分野から複数選択)
- 難易度フィルタ (易/中/難)
- 出題条件 (未回答のみ / 直近誤答のみ / SRS推奨のみ / ブックマーク済のみ)
- 出題数 (10/20/50/100問または無制限)

**処理フロー**

- フィルタ条件に基づき問題プールを構築
- SRS推奨モードの場合、復習期日順にソート
- 1問ずつ表示し、選択肢クリックで即判定
- 正誤・回答時間・選択した選択肢を学習履歴に記録
- 解説・参考リンク・関連用語を表示
- 理解度を4段階で自己評価 (Again/Hard/Good/Easy)
- SRSアルゴリズムで次回出題日を計算・更新

**画面要件**

- スマホ片手操作対応 (通勤利用想定)
- 1タップで選択肢回答
- スワイプで次問題へ遷移

#### F-03: 模擬試験モード

**概要**: 本番試験を再現した時間制限付き145問テスト

| 項目 | 仕様 |
| --- | --- |
| 問題数 | 145問 (本番準拠) |
| 制限時間 | 100分 (本番準拠) |
| 出題 | シラバス分野の出題比率を本番想定に合わせて自動構成 |
| 中断・再開 | 不可 (本番再現のため一気通貫) |
| タイマー | 残り時間タイマー常時表示、1問あたり推奨時間表示 |
| スキップ | 問題スキップ・後で見直しフラグ機能 |
| 採点 | 全問解答後に一括採点 |

**結果表示**

- 総得点・合格ライン (70%) との対比
- 分野別正答率
- 時間配分分析 (1問あたり平均/最大時間)
- 誤答問題の一覧と解説への直リンク
- 過去模試との比較推移グラフ

#### F-06: 間隔反復学習 (SRS)

**概要**: Anki/SuperMemo方式の記憶定着アルゴリズム

**アルゴリズム**: SM-2 (SuperMemo 2) 方式またはAnki派生

**動作詳細**

- 各問題に対して「次回復習日」「易しさ係数(EF)」「連続正答回数」を保持
- 自己評価4段階で次回間隔を計算
  - Again (0): 当日中に再出題、EF低下
  - Hard (1): 1日後、EF微減
  - Good (2): 標準間隔(前回×EF)、EF維持
  - Easy (3): 拡大間隔(前回×EF×1.3)、EF増加
- 毎日の「今日の復習問題数」をダッシュボードに表示

#### F-10: Claude APIによる解説生成

**概要**: Anthropic社のClaude APIを利用し、問題に対する解説文を自動生成する。

**利用モデル**: Claude Haiku 4.5

- 理由: 解説生成は定型的なタスクで、コスパ最重視のためHaiku 4.5を選択
- 価格: 入力 $1.00 / 出力 $5.00 per 1M tokens
- 1解説あたり想定: 入力500トークン + 出力800トークン ≒ $0.0045 (約0.7円)
- 700問すべてに解説生成しても約500円程度

**動作**

- 問題と正解、シラバス分野をプロンプトに含めてClaude APIに送信
- 既存解説が不十分な問題に対して補完説明を生成
- 生成結果はDBにキャッシュ (毎回生成しない)
- 自分で内容確認後に正規解説として採用可
- プロンプトキャッシュを活用 (システムプロンプト共通部分は90%割引)

**プロンプト設計例**

```
system: あなたはG検定対策の優秀な講師です。
        ディープラーニング・AI倫理・法律に精通しており、
        受験者が暗記でなく理解できる解説を提供します。

user: 【問題】{question}
      【正解】{correct}
      【分野】{category}
      この問題の解説を300字程度で書いてください。
      正解の理由・関連知識・覚え方のコツを含めてください。
```

#### F-11: Claude APIへの質問機能 (RAGチャット)

**概要**: 個別の問題や用語について、Claudeに自由に質問できるチャット機能。

**利用モデル**: Claude Sonnet 4.6

- 理由: 質問応答はより高い理解力が必要。Sonnet 4.6が品質とコストのバランスが最良
- 価格: 入力 $3.00 / 出力 $15.00 per 1M tokens
- 1質問あたり想定: 入力2,000トークン (RAGコンテキスト含む) + 出力500トークン ≒ $0.014 (約2円)
- プロンプトキャッシュで実質コストはさらに低減

**動作**

- 問題画面右下に「Claudeに質問」ボタン
- 当該問題の文脈をプロンプトに含めて送信
- RAG: 公式テキスト要約・シラバス・過去解説をpgvectorで類似検索し、関連情報をプロンプトに付与
- 会話履歴は問題ごとに保持 (最大10ターン、それ以降は要約圧縮)
- レスポンスはストリーミング表示 (Server-Sent Events)

#### F-19: Claude APIで問題を生成する機能

**概要**: 自分のシラバス進捗に合わせて、不足分野の問題をClaudeに生成させる。

**利用モデル**: Claude Sonnet 4.6

- 理由: 問題作成は質が重要。トリッキーで本番同様の問題を生成するにはSonnet 4.6が必要
- 生成後は必ず自分で内容確認・修正してからDB登録 (誤答含み問題の混入防止)

**動作**

- 分野・難易度・問題形式を指定してリクエスト
- Claudeが問題文・選択肢・正解・解説を生成
- 管理画面で内容確認・編集後、DBに登録
- API呼び出しコストはF-20で監視

#### F-20: API利用コスト表示・上限アラート

**概要**: Claude APIの利用コストを可視化し、予算超過を防ぐ。

**動作**

- API呼び出しごとに利用トークン数・推定コストをDBに記録
- ダッシュボードに「今月のAPI利用額」を表示
- 月額上限 (デフォルト$10 = 約1,500円) 設定
- 上限の80%到達でメール通知、100%到達でAPI呼び出し停止
- Anthropic Console側のSpend Limit機能と二重で防御

### 3.3 セキュリティ要件

個人利用なので最小限の実装に絞る。

- HTTPS強制 (Let's Encrypt自動更新)
- 認証: シングルユーザーのため自分専用ログイン (BASIC認証 or シンプルJWT)
- パスワード: bcrypt (cost=12) でハッシュ化
- Claude API キー: 環境変数で厳重に管理 (.envはgit管理外)
- API キーをフロントエンドに絶対渡さない (バックエンド経由のみ)
- レート制限: API 30リクエスト/分 (自分専用なので緩め)
- Anthropic Console側でSpend Limit設定 (月$15上限)
- ファイアウォール: ConoHa VPS標準 + ufw (22/80/443のみ開放)
- SSH: 公開鍵認証必須・パスワード認証無効化・rootログイン禁止
- fail2ban導入
- CSRF・XSS対策: フレームワーク標準機能で対応

### 3.4 運用要件

- バックアップ: PostgreSQLを毎日2:00 JSTにダンプ、7世代保持
- バックアップ先: ローカル＋自宅PCへrsync (オブジェクトストレージは個人利用のため不要)
- 監視: Uptime Kuma (シンプルな死活監視のみ)
- アラート: Discord/メール (Down検知時、API予算超過時)
- ログ: journald + logrotate (90日保持)
- デプロイ: GitHub Actions or 手動 (個人利用なのでどちらでも可)
- Claude APIログ: 月次でコスト確認、不要な呼び出しがないか点検

### 3.5 保守性・拡張性

- コード管理: GitHub (privateリポジトリ)
- ドキュメント: README.md, CLAUDE.md (Claude Code開発用)
- API設計: OpenAPI 3.0準拠 (FastAPI自動生成)
- テスト: pytest (BE) / Vitest (FE) / カバレッジ50%以上 (個人利用なので緩め)
- CI: GitHub Actionsでpush時にテスト・lint実行

### 3.6 対応ブラウザ・デバイス

- PC: Chrome / Edge 最新版 (山城さんメイン環境)
- スマホ: iOS Safari / Android Chrome 最新版
- 画面サイズ: 320px～4Kまでレスポンシブ
- 最重要: スマホ縦持ち (通勤利用想定)

### 4.2 Claude API連携の設計

#### 利用モデルと使い分け

| 用途 | モデル | 理由 | 想定単価 |
| --- | --- | --- | --- |
| 問題解説の自動生成 (F-10) | Claude Haiku 4.5 | 定型的・大量生成・コスパ重視 | $0.0045/解説 |
| 質問応答チャット (F-11) | Claude Sonnet 4.6 | RAG含む高度な質問応答に必要な品質 | $0.014/質問 |
| 問題自動生成 (F-19) | Claude Sonnet 4.6 | 問題の質が学習効果に直結するため | $0.02/問題 |

#### API キー管理

- Anthropic ConsoleでAPIキーを取得 (https://console.anthropic.com)
- 環境変数 `ANTHROPIC_API_KEY` としてサーバに保管
- `.env` ファイルに記載しgitignoreに追加
- 本番サーバではsystemd EnvironmentFile経由で読み込み
- フロントエンドには絶対に露出させない (バックエンドプロキシ経由)
- 万一漏洩した場合に備え、Anthropic Console側でSpend Limitを月$15に設定

#### コスト最適化策

- **プロンプトキャッシュ活用**: システムプロンプト・固定コンテキストはキャッシュ (90%割引)
- 解説生成結果は必ずDBに永続化 (再生成しない)
- 質問応答はセッション単位で会話履歴を要約圧縮 (10ターン超過時)
- **Batch API活用**: 解説の一括生成時は50%割引
- 月額上限を$10に設定 (自分用なので十分)

#### API呼び出し実装 (Pythonサンプル)

```python
from anthropic import Anthropic

client = Anthropic()  # ANTHROPIC_API_KEY環境変数から自動読込

def generate_explanation(question, correct, category):
    response = client.messages.create(
        model="claude-haiku-4-5",
        max_tokens=1024,
        system=[{
            "type": "text",
            "text": SYSTEM_PROMPT,
            "cache_control": {"type": "ephemeral"}
        }],
        messages=[{
            "role": "user",
            "content": f"問題: {question}\n正解: {correct}\n分野: {category}"
        }]
    )
    # コスト記録
    log_api_usage(response.usage, "claude-haiku-4-5", "explanation")
    return response.content[0].text
```

## 8. リスク管理

| データ消失 | 高 | 日次バックアップ、複数世代保持、自宅PCにrsync |
| Claude API回答の不正確さ | 中 | AI生成内容は必ず自分で検証、公式テキストでクロスチェック |
